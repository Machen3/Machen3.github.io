HTML和XPath以及CSS selector

HTML又叫HyperText Markup Language，常和CSS以及JaveScript一起被用于设计网站。作为一种结构和标记语言使用。
在python中常用bs4对网页进行解析，支持HTML和CSS。

  • 第一層是 <html>...</html>
  • 第二層是 <head>...</head>與 <body>...</body>
  • 第三層是 <title>...</title> 、<h1>...</h1> 與三組 <p>...</p>

用法：

除了文字信息外，我们还可能要求获取标记中的属性

  • id 用來標註不同的<div>...</div> 、<li>...</li>
  • class 用來標註三組<li>...</li>
  • href 是 <a>...</a>的連結網址屬性

    用法(soup.find(id="movie-3").find("a").get("href"))

XPath又叫XML Path Language与CSS一起常用来定位HTML档案中元素。
在python中bs4并不支持XPath语言，但是其他常用解析库比如lxml、Selenium和Scrapy等都支持该语言。

XPath的基本组成

XPath的4种常见定位方法

  • 根节点与非根节点
    • /div选择div节点（当div为根节点的时候），理解为在树状结构中向下移动一层。
    • //div选择所有的div节点，包括非根节点，理解为在树状结构中的所以层寻找指定标记。
  • 通过属性选择节点
    • @用来指定属性，比如id \ class\ href等。
    • //@href
    • //a[@href = 'http://google.com']
  • 通过位置选择
    • //[3]选择第三个
    • //table[last()]选择最后一个表
    • a[position()<3]选择前三个
  • 配合*匹配任意字符和节点
    • //table/tr/*
    • //div[@*]

结论：

1.两者都是数据处理工具,Pandas在大量数据运算时候非常有优势,单纯对比下Excel可视化程度更高。
2.如果是一次性操作，Excel实现起来更快,因为很多操作都黑箱化了.不过对Pandas利用熟练或者有封装好的函数也可以实现
3.最重要的是Pandas可以程序化重复使用，虽然Excel也可以。
4.Pandas的可视化和数据展示得结合其他库的使用，这个后续有待开发
